QuantPulse AI — Automated BTC/USDT Trading System

Algorithmic Trading & ML Ensemble System
▼
Problem Statement:

Build a fully automated BTC/USDT trading advisory system that fetches live market data from Binance, applies a 5-model ML ensemble with 12 professional trading rules, and delivers real-time Buy/Sell/Hold recommendations with exact Entry, Stop Loss, and Take-Profit levels — every 3 minutes, 24/7.

Data Pipeline:
  • Live OHLCV candle data from Binance REST API across 3 timeframes (3m, 5m, 15m)
  • 1,000 candles per request with full timestamp validation
  • SQLite storage for historical training and live inference
  • Background scheduler auto-refreshes predictions every 3 minutes
Feature Engineering (40+ Features):
  • Price Action: body size, wick ratios, high-low range, price change %
  • Momentum: RSI (7/14/21), MACD histogram & cross signals, slope/lag features
  • Volatility: ATR percentile (7/14), Bollinger Band position & width
  • Volume & Orderflow: taker buy ratio, delta volume, volume lag 1/2/3
  • Market Structure: Higher-High/Lower-Low flags, compression flag, ADX strength
ML Ensemble (5 Models):
  • XGBoost — gradient boosting for price pattern recognition
  • LightGBM — fast boosting optimized for volume & momentum features
  • CatBoost — handles categorical regime features with high accuracy
  • Random Forest — noise-resistant ensemble base model
  • Meta-Model (Logistic Regression) — stacks all 4 base models into final prediction
  • Walk-Forward validation with 0.6% barrier labeling to filter noisy micro-moves
  • Mutual Information feature selection — top 25 from 80+ engineered features
Signal Engine (12 Professional Trading Rules):
  • Multi-Timeframe (MTF) Confirmation: 15m sets trend, 5m confirms, 3m times entry
  • 9-Point Trade Filter: ADX > 20, Volume Ratio > 0.7, RSI 15–85, Regime ≠ VOLATILE
  • ATR-Based Levels: SL at −1× ATR, TP1 at +1×, TP2 at +1.8×, TP3 at +3× ATR
  • Support/Resistance detection, EMA touch entry, signal smoothing via majority vote
  • Regime-scaled position sizing with partial exit strategy (50% / 30% / 20%)
Model Evaluation:
  • Walk-Forward validation across multiple time-ordered folds (no data leakage)
  • Regime-specific models: separate training for TRENDING vs RANGING markets
  • PnL simulation: win rate and P&L tracked over last 30 live predictions
  • Dynamic confidence threshold (50–65%) auto-adjusted by volatility percentile
Business Impact: End-to-end automated system eliminates manual chart analysis. 5-model ensemble with 12-rule signal filter significantly reduces false signals. Real-time Flask dashboard with Groq LLM chatbot (Llama-3.3-70B) delivers instant trade advisory — zero zero intervention needed, running 24/7.
Deployment:
  • Flask REST API with 10+ endpoints (/api/predictions, /api/mtf_signal, /api/btc_trend, /api/pnl)
  • Real-time web dashboard: live BTC price ticker, per-TF signal cards, MTF alignment panel, 9-point filter display
  • Groq LLM AI chatbot — auto-sends full market context every 3 minutes
  • Redis/in-memory cache with TTL + background auto-retrain scheduler
  • Dual API key fallback system for 24/7 uninterrupted operation

Credit Card Fraud Detection System

Fintech & Security
▼
Problem Statement:

Detect fraudulent credit card transactions in real-time under extreme class imbalance (<0.2% fraud) without excessive customer friction.

Dataset & Features:
  • Credit Card transactions dataset (~284k transactions)
  • Extreme class imbalance: <0.2% fraud rate
  • PCA-transformed features + custom engineering
  • Feature engineering: log_amount, time-of-day, transaction velocity, interaction features
Modeling Approach:
  • Logistic Regression (baseline for imbalanced data)
  • Random Forest (ensemble approach)
  • XGBoost (selected for superior recall-precision trade-off)
Handling Class Imbalance:
  • SMOTE for synthetic sample generation
  • Stratified cross-validation
  • Recall-focused evaluation metrics
Threshold Optimization:
  • Default threshold 0.5 → Tuned threshold based on business costs
  • False Negative cost >> False Positive cost (missed fraud >> false alarms)
  • Result: Reduced missed fraud by 60%, recall ~91%
Business Impact: Advanced fraud detection minimizes financial loss, protects customer assets, reduces chargeback costs, and maintains customer trust without excessive false positive friction.
Deployment:
  • FastAPI backend for real-time fraud scoring
  • Streamlit frontend for interactive prediction exploration
  • Sub-second latency for transaction approval workflow
