  • API-ready for integration with lending platforms
  • Risk scoring for real-time customer assessment

House Price Prediction with Advanced Feature Engineering

Real Estate & Regression Modeling
▼
Problem Statement:

Predict house prices with high accuracy while addressing underprediction bias in the tail (expensive properties) for real estate valuation.

Dataset & Features:
  • Kaggle House Prices dataset (79 features, ~1460 records)
  • Advanced feature engineering: log transforms, interaction terms, total area calculations
  • Handling missing values and categorical encoding
Modeling Approach:
  • Linear Regression (baseline)
  • Ridge & Lasso (regularization comparison)
  • XGBoost (selected for robust predictions)
Key Debugging & Improvements:
  • Identified 26% underprediction error in tail risk (expensive homes)
  • Fixed by using raw SalePrice target (no log transforms for target)
  • Disabled Lasso's aggressive shrinkage, increased XGBoost tree depth
  • Applied robust regularization for stability
Model Evaluation:
  • MAE: 0.88% (excellent accuracy)
  • Tail-risk analysis for expensive properties
  • Stratified cross-validation RMSE verification
Business Impact: Tail-risk-aware regression improves real-world deployment reliability, ensures accurate pricing for premium properties, and supports fair market valuations.
Deployment:
  • FastAPI inference service with real-time pricing
  • Partial input handling for missing features
  • Robust schema alignment for production data

Credit Card Fraud Detection System

Fintech & Security
▼
Problem Statement:

Detect fraudulent credit card transactions in real-time under extreme class imbalance (<0.2% fraud) without excessive customer friction.

Dataset & Features:
  • Credit Card transactions dataset (~284k transactions)
  • Extreme class imbalance: <0.2% fraud rate
  • PCA-transformed features + custom engineering
  • Feature engineering: log_amount, time-of-day, transaction velocity, interaction features
Modeling Approach:
  • Logistic Regression (baseline for imbalanced data)
  • Random Forest (ensemble approach)
  • XGBoost (selected for superior recall-precision trade-off)
Handling Class Imbalance:
  • SMOTE for synthetic sample generation
  • Stratified cross-validation
  • Recall-focused evaluation metrics
Threshold Optimization:
  • Default threshold 0.5 → Tuned threshold based on business costs
  • False Negative cost >> False Positive cost (missed fraud >> false alarms)
  • Result: Reduced missed fraud by 60%, recall ~91%
Business Impact: Advanced fraud detection minimizes financial loss, protects customer assets, reduces chargeback costs, and maintains customer trust without excessive false positive friction.
Deployment:
  • FastAPI backend for real-time fraud scoring
  • Streamlit frontend for interactive prediction exploration
  • Sub-second latency for transaction approval workflow
